# animator-clip 发布流程

> **核心规则**：Step 1-7 每步执行前必须向用户确认，获得确认后再执行。任意步骤出错则停止并告知原因，等待用户指示。

---

## Step 1 — 更新版本号

读取 `package.json` 中的当前版本号告知用户，询问本次发布的目标版本号。用户给出版本号后，向用户确认，确认后用 Edit 工具更新 `version` 字段，完成后告知用户。

---

## Step 2 — 打包源码

确认后执行 `npm run lib`，成功后告知用户，失败则展示错误信息并停止。

---

## Step 3 — 更新并检查文档

确认后，读取 `src/` 所有源码，逐一对照 `docs/` 下的所有 `.md` 文件，检查并修正：

- API 描述（参数名、类型、返回值）是否与源码一致
- 示例代码是否正确（语法、方法名、属性名）
- 是否存在拼写错误、注释错误、描述错误
- 是否存在源码里没有的虚构 API 或链接

**严格基于源码进行更新，不得编造任何语法、方法或链接。**

完成后汇报修改了哪些文件及具体问题，若无问题则说明文档已是最新。

---

## Step 4 — 更新 Changelog

确认后，读取 `docs/changelog.md` 的现有格式，在文件顶部新增本次版本条目，格式与已有条目保持一致，仅描述本次实际存在的变更。若无法判断具体改动，写入版本标题后留空，并提示用户手动补充。完成后将新增内容展示给用户。

---

## Step 5 — 打包文档

确认后执行 `npm run docs:build`，成功后告知用户，失败则展示错误信息并停止。

---

## Step 6 — 更新 README

确认后，读取 `src/` 源码、`docs/` 文档和现有 `README.md`，修正 README 中与最新源码不符的内容（API、参数、示例等）。

**严格基于源码进行更新，不得编造任何语法、方法、功能描述或链接。**

完成后告知用户。

---

## Step 7 — 发布到 npm

确认后执行 `npm publish`，成功后告知用户。失败时展示完整错误、分析原因（未登录、版本冲突、网络等），根据用户指示处理后重试。

---

## 完成汇报

所有步骤完成后自动输出总结：

```
发布完成！

库名：animator-clip
版本：X.X.X
npm：https://www.npmjs.com/package/animator-clip

✅ Step 1 — 版本号更新至 X.X.X
✅ Step 2 — 源码打包完成
✅ Step 3 — 文档审查与更新完成
✅ Step 4 — Changelog 更新完成
✅ Step 5 — 文档打包完成
✅ Step 6 — README 更新完成
✅ Step 7 — npm 发布成功
```
